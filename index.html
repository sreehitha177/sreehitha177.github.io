GitHub Pages Setup Guide

Fix the 404 error and successfully deploy your portfolio website

Why am I seeing a 404 error?

A 404 error on GitHub Pages usually means that GitHub cannot find your website files. This typically happens when:

  • Your repository is not configured for GitHub Pages
  • Your files are not in the correct branch
  • You don't have an index.html file in the root directory
  • Your repository is private (GitHub Pages requires public repos for free accounts)
Note: After setting up GitHub Pages, it can take up to 10 minutes for your site to become available.

Step-by-Step Fix

1

Create a new repository

Name it username.github.io (replace "username" with your GitHub username)

2

Upload your files

Make sure you have an index.html file in the root of your repository

3

Configure GitHub Pages

Go to Settings ‚Üí Pages ‚Üí Branch, select "main" (or "master") and click Save

4

Wait for deployment

It can take a few minutes for your site to be published. Refresh after 5-10 minutes.

Success: Your site should now be available at https://username.github.io
